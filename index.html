    07 · Internal mechanisms

    The architecture’s
    load-bearing details.

    Synchronous critical section

    Why a threading.RLock exists inside an async library

    Executor.submit() is synchronous, while done callbacks can execute on worker or manager threads. The lock makes lease validation, submission, pending-future registration, release, and callback accounting one coherent state transition.

    Same lease contract.
    Different execution physics.

    Python process
    threadThreadPoolExecutor

    Best for blocking I/O, synchronous SDKs, file access, and small blocking operations.

    • Shared process memory
    • No pickle boundary
    • User code must be thread-safe
    IPC
    processProcessPoolExecutor

    Best for CPU-heavy Python work that benefits from separate processes and CPU cores.

    • Picklable inputs and outputs
    • Process startup and IPC cost
    • Optional child-log forwarding

    interpreterInterpreterPoolExecutor

    Python 3.14+ backend exposed through the same build and leasing hook.

    • Version-gated availability
    • Thread-name prefix support
    • Same retirement semantics
    Process log forwarding

    Child records cross a queue; application logging stays in charge.

    The manager composes its worker initializer with the user’s initializer, installs a QueueHandler in children, and forwards records through a parent QueueListener.

    11 · Canonical usage

    Lease capacity around
    the synchronous boundary.

    Start and stop on one event loop. Keep the lease scope explicit. Let the context manager release after the result resolves.

    1manager owns executor lifetime2lease owns submission rights3caller owns task semantics
    app.py
    from leasepool import ExecutorBackend, LeasedExecutorManager

manager = LeasedExecutorManager(
    backend=ExecutorBackend.THREAD,
    max_pools=3,
    min_pools=1,
    workers_per_pool=4,
)

await manager.start()
try:
    async with await manager.acquire(
        owner="vendor-sdk",
        lease_seconds=30,
    ) as lease:
        result = await lease.run(sync_call, payload)
finally:
    await manager.stop()
    12 · Architectural invariants

    The rules that keep
    the system honest.

    01

    Raw executors stay private.

    Callers receive a proxy that cannot shut down managed capacity.

    02

    Release is a submission barrier.

    New work is rejected immediately, even when old futures are still draining.

    03

    Hard expiry is terminal.

    An expired executor is shut down and never recycled into available capacity.

    04

    Broken means retired.

    A backend-level failure changes pool topology; an ordinary task error does not.

    05

    Loop ownership is explicit.

    Manager and grinder async APIs remain on the loop that started them.

    06

    Shutdown never self-joins.

    Callback-triggered executor shutdown is deferred beyond backend-owned threads.
